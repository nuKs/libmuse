Detailed Description

A IXNMuse associated with a IXNMuseFileReader .

This interface can be used to play back packets from a .muse file so that they can be handled the same way that you would handle data from an actual IXNMuse . Details of the parsing are abstracted and you will simply receive data packets with the data that is read.

When you use this class, register IXNMuseDataListener for the data packets you would like to receive by calling asMuse (IXNReaderMuse): then registerDataListener:type: (IXNMuse).

Downcast to IXNMuse Note that most of the API of the returned Muse is stubbed and will cause assert failures if called in debug mode. The methods that are implemented are:

  • isLowEnergy / is_low_energy
  • enableDataTransmission / enable_data_transmission

Note that the only listener that will ever receive packets from a IXNReaderMuse is IXNMuseDataListener .

§ playback()

- (void) playback

Play back the IXNMuseFile and call each listener for each packet as appropriate.

The speed of the playback is controlled by the playback settings. Simulated playback settings require the IXNReaderMuse to be constructed with an IXNEventLoop to simulate the time between packets. Calling playback without an IXNEventLoop will not playback the file and will log a warning.

This only works once; the file is not rewound on subsequent calls. If you want to play a file again, create another IXNReaderMuse

IXNReaderMusePlaybackSettings
